# Dr. Solution.

**Dr. Solution.** منصة أكاديمية لتحويل الكتب والأبحاث والمحاضرات والمستندات الجامعية إلى تجربة دراسة متكاملة. تُحوّل كل صفحة إلى سلايد بصري، ثم تضيف الترجمة والشرح وتحليل المحتوى ووصف الصور والخلاصة، مع اختبار اختياري يحدد المستخدم نوعه وصعوبته.

## ما الذي تنتجه المنصة؟

- شرح مستقل لكل صفحة مع صورة الصفحة الأصلية.
- ترجمة المحتوى وتحليل الرسوم والجداول والصور.
- نقاط أساسية وخلاصة لكل سلايد.
- أهداف تعلم وخلاصة ومصطلحات للمادة كاملة.
- أسئلة اختيار متعدد، صح أو خطأ، إجابة قصيرة، أو مقالية.
- ملف HTML واحد تفاعلي يعمل دون خادم بعد تنزيله.

يدعم أساليب مخصصة للمحاضرات، الأبحاث العلمية، الكتب، والمستندات الجامعية.

تستخدم الواجهة العامة `Gemini 3.1 Flash-Lite` لأنه يدعم النصوص والصور ضمن طبقة Google المجانية. تبقى الحدود اليومية وحدود الدقيقة مرتبطة بمشروع كل مستخدم. DeepSeek API مدفوع حسب الاستهلاك، لذلك لا يظهر ضمن الوضع المجاني.

## التشغيل المحلي

يتطلب Python 3.11 أو أحدث.

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

يمكنك إضافة مفاتيحك داخل `.env` للاختبار المحلي، أو تركها فارغة وإدخال المفتاح من الواجهة. ثم شغّل:

```powershell
python run.py
```

افتح `http://127.0.0.1:8000`.

## الإعدادات

- `GEMINI_API_KEY`: مفتاح Google AI Studio للخيار المجاني الافتراضي.
- `GEMINI_MODEL`: النموذج الافتراضي `gemini-3.1-flash-lite`.
- `ANTHROPIC_API_KEY`: اختياري لربط Claude.
- `DEEPSEEK_API_KEY`: تكامل خلفي اختياري ومدفوع، وغير ظاهر في الواجهة المجانية.
- `REQUIRE_USER_API_KEY`: عند ضبطه على `true` يجب أن يضع كل مستخدم مفتاحه في الواجهة.
- `MAX_UPLOAD_MB`: أقصى حجم للملف، والافتراضي 15 MB.
- `MAX_PAGES`: أقصى عدد صفحات في العملية، والافتراضي 24.
- `ANALYSIS_BATCH_SIZE`: عدد الصفحات في طلب التحليل الواحد، والافتراضي 4.

في وضع BYOK يُرسل مفتاح المستخدم عبر HTTPS مع طلب التحليل فقط، ويُستخدم في ذاكرة الخادم ثم يُهمل. لا يُحفظ في السجل أو IndexedDB أو ملف HTML الناتج. لا ترفع `.env` إلى GitHub.

## النشر على Render

1. ارفع المشروع إلى مستودع GitHub.
2. اختر **New Blueprint** في Render وحدد المستودع.
3. سيقرأ Render ملف `render.yaml` ويشغّل وضع BYOK تلقائياً؛ لا يحتاج مفتاحاً مركزياً.
4. كل مستخدم يحصل على مفتاح Gemini أو DeepSeek من الرابط الظاهر في المنصة ويضعه قبل التحليل.
5. افحص `/api/health` بعد النشر؛ يجب أن تكون الحالة `ok`.

يدعم المشروع أيضاً Docker للنشر على Railway أو أي منصة تدعم الحاويات.
